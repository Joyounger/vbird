Title: [鸟哥的 Linux 私房菜 -- 基础学习篇] “Linux 账号管理与ACL 权限控制”当中myuser1的visudo配置 - 存在安全隐患, 附上个人的一点建议

原配置如下：
[root@www ~]# visudo <==注意是 root 身份
myuser1 ALL=(root) !/usr/bin/passwd, /usr/bin/passwd [A-Za-z]*, \
!/usr/bin/passwd root

在原配置的情况下，经过检验，以下操作可更改、封锁、解锁、过期root的密码
# 1. 更改root的密码
[myuser1@centos ~]$ sudo passwd root --stdin
# 2. 对root（或其他用户）进行密码封锁、解锁、过期，更改密码最大最小生命周期
[myuser1@centos ~]$ sudo passwd root [-lued] [-n mindays] [-x maxdays]\
[-w warndays] [-i inactivedays] [-S]

建议把配置修改为如下，防止root后面存在任何参数的可能
[root@www ~]# visudo <==注意是 root 身份
myuser1 ALL=(root) !/usr/bin/passwd, /usr/bin/passwd [A-Za-z]*, \
!/usr/bin/passwd root*
或
[root@www ~]# visudo <==注意是 root 身份
myuser1 ALL=(root) /usr/bin/passwd [A-Za-z]*, !/usr/bin/passwd root*
当中把'!/usr/bin/passwd, '删除，因为后面的[A-Za-z]*在bash环境中表示至少会有一个字母，
即把前者也否定掉了。
PS: 通过`sudo passwd <not-root-user> [-lued..]',其实myuser1除了更改用户密码，
也可以对用户进行passwd的其他操作，不过目前感觉这不会是一个重要的问题。
---
同时 接着的内容“透过别名建置 visudo：”也应该作出相应的修改～
如确定需要更改，也请同时在繁体简体都作出相应更新，方便读者:)

From Linux Beginner
